## 1. Какво е тази система

AI асистент за хотелски мениджмънт, достъпен чрез **Telegram**. Пишеш или говориш — получаваш отговор с реални данни от Google Sheets.

**Може:**
- Заетост, ADR, RevPAR за конкретна дата или период
- Резервации, пристигания, заминавания
- Графици, смени, заплати на персонала
- Фактури, разходи, приходи от бар
- Инвентар, закуски, food cost
- Стаи, леглодни, housekeeping
- **НОВО:** Цени на конкуренцията (Google Hotels)
- **НОВО:** Ревюта и рейтинги от Google Maps
- **НОВО:** Метео за всеки град в България
- **НОВО:** Полети до/от София, Бургас, Варна
- **НОВО:** Пътна обстановка в България
- Сутрешен брифинг (автоматичен в 07:00)
- Нощен одит (автоматичен в 21:00)

## 2. Бърз старт

### Стъпка 1: Отвори Telegram бота
Изпрати съобщение на бота (трябва да си в списъка с одобрени потребители).

### Стъпка 2: Питай каквото искаш
Примери:

| Въпрос | Какво получаваш |
|--------|-----------------|
| „Сутрешен брифинг" | Пълен GM Morning Brief |
| „Заетост за днес" | Occupancy, ADR, RevPAR |
| „Какви са цените на хотелите в Банско за 25-26 март?" | Цени на конкуренцията от Google |
| „Какъв е рейтингът на Кемпински Банско?" | Google ревюта и рейтинг |
| „Какво е времето във Варна?" | Метео прогноза |
| „Полети пристигащи в София" | Реални полети от AviationStack |
| „Графикът за тази седмица" | Смени на персонала |
| „Фактури за март" | Разходни документи |

### Стъпка 3: Говори вместо да пишеш
Изпрати **гласово съобщение** — системата го транскрибира и обработва.

## 4. Агенти и отговорности

Системата има **1 главен координатор** и **6 специалисти**:

| Агент | Роля | Данни |
|-------|------|-------|
| **Hotel Manager** | Координатор — разбира въпроса, разпределя, синтезира | Всички (чрез sub-agents) |
| **Front Office** | Заетост, резервации, рецепция | Forecast, Reservations, Reception, Payments |
| **HR** | Графици, заплати, персонал | Schedule, Salary, Forecast |
| **Accounting** | Фактури, разходи, бар приходи | Expenses, Payments, Bar Income |
| **F&B** | Инвентар, закуски, food cost | Inventory, Forecast, Payments |
| **Housekeeping** | Стаи, леглодни, OOS | Forecast (само една дата) |
| **Marketing** | Конкуренция, цени, ревюта | Google Hotels, Google Reviews, Forecast |

### Директни tools на координатора
- **get_weather** — метео за всеки град в България
- **flights_bulgaria** — полети SOF, BOJ, VAR
- **road_conditions** — пътна обстановка

## 6. Нови в V10

### Конкурентен анализ
Marketing Manager вече може да проверява **реални цени** на хотели в Банско (и други градове) чрез Google Hotels:
- Цени в BGN за конкретни дати
- Рейтинг и брой ревюта
- Сравнение с вашия ADR

**Примерни въпроси:**
```
Какви са цените в Банско за Великден?
Топ 5 най-евтини хотели за 25-26 март?
Рейтинг на Premier Luxury Mountain Resort
Сравни цените ни с конкуренцията
```

### Времe за цяла България
Вече не е само Банско — питай за всеки град:
```
Времето в Пловдив?
Каква е температурата в Бургас?
```

### Полети
Реални данни за полети до/от българските летища:
- **SOF** — София
- **BOJ** — Бургас
- **VAR** — Варна

```
Какви полети пристигат в София?
Заминаващи от Варна днес?
```

### Пътна обстановка
```
Каква е пътната обстановка?
Има ли затворени пътища?
```

## 7. Как работи отвътре

```
Ти пишеш: „Заетост за утре"
                ↓
         Telegram Trigger
                ↓
         Access Control ← Проверка дали си одобрен
                ↓
         Set Node ← Добавя [CONTEXT: Дата: 19.03.2026, Ден: Wednesday, Час: 17:30]
                ↓
         Hotel Manager (GPT-4o) ← Разбира, че трябва Front Office
                ↓
         Front Office Manager (GPT-4o-mini) ← Вика Forecast_Tool
                ↓
         Google Sheets ← Връща: Заетост 78%, ADR 125 BGN, RevPAR 97.50 BGN
                ↓
         Hotel Manager ← Синтезира отговора на български
                ↓
         Telegram ← Получаваш отговор
```

## 8. Ограничения

### DATA_GAP
Когато данни липсват, системата го казва ясно:
```
DATA_GAP — няма свързан източник за Google Ads ROI
```
Това е по-добре от измислени данни.

### Текущи ограничения
| Ограничение | Детайли |
|-------------|---------|
| Google Ads / Meta ROI | Не е свързан — Marketing връща DATA_GAP |
| Housekeeping | Само за **една дата** — не поддържа периоди |
| SerpAPI лимит | 100 търсения/месец за Google Hotels + Reviews |
| AviationStack лимит | 100 заявки/месец за полети |
| Road conditions | API може да не е публично достъпен — нуждае се от тестване |

| Spreadsheet | Какво съдържа | Sheet |
|-------------|---------------|-------|

3. Промени условието от `[425406605]` на `[425406605, НОВОТО_ID]`

## 11. Технически setup

### Изисквания
- n8n Cloud акаунт (или self-hosted)
- OpenAI API ключ (GPT-4o + GPT-4o-mini)
- Google Sheets API credentials
- Telegram Bot Token
- Redis instance
- OpenWeatherMap API ключ
- SerpAPI ключ (за конкурентен анализ)
- AviationStack API ключ (за полети)

### Ключови файлове
| Файл | Описание |
|------|----------|
| `Hotel General Manager Multi V10 Stable.json` | Import в n8n |
| `aggregator.gs` | Deploy като Google Apps Script Web App |

### Deploy на aggregator.gs
1. Отвори https://script.google.com
2. Създай нов проект, постави кода от `aggregator.gs`
3. Deploy → New deployment → Web app → Anyone can access
4. Копирай URL-а и го постави в `Forecast_Aggregated_Tool` node

## 12. Troubleshooting

| Проблем | Решение |
|---------|---------|
| „Възникна грешка при обработката" | Провери n8n executions за error detail |
| Дати не се показват правилно | Датата се инжектира чрез chatInput — провери Set nodes |
| Context overflow | Провери Redis memory limit (трябва да е 10) и sheet ranges |
| Google Sheets 429 error | Retry logic го покрива — изчакай 1 минута |
| „Нямате достъп" | Telegram ID не е в Access Control whitelist |
| Forecast_Aggregated_Tool гърми | Провери дали URL е хардкоднат (не $env) |
| Competitor data не работи | Провери SerpAPI лимит (100/месец) |
| Полети не работят | Провери AviationStack лимит (100/месец) |
